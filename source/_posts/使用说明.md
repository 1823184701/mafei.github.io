本博客基于 **Hexo** 框架 + **Fluid** 主题搭建，部署于 **GitHub Pages**。本文介绍博客的使用方法，方便你快速上手。

<!-- more -->

## 文章存放位置

所有文章以 Markdown 格式存放在 `source/_posts/` 目录下。

## 创建新文章

**方式一：命令行**
```bash
npx hexo new post "文章标题"
```
会自动生成带 Front-matter 的 `.md` 文件。

**方式二：手动创建**
在 `source/_posts/` 下新建 `.md` 文件，需包含 Front-matter。

## 文章格式示例

```markdown
---
title: 文章标题
date: 2026-05-21 23:30:00
tags:
  - 标签1
  - 标签2
categories:
  - 分类名称
cover: /img/cover.jpg        # 封面图（可选）
excerpt: 文章摘要文字          # 摘要（可选）
---

正文内容，支持完整 Markdown 语法。

## 二级标题

## Front-matter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 文章标题 |
| `date` | ✅ | 发布日期 |
| `updated` | ❌ | 更新日期 |
| `tags` | ❌ | 标签（数组格式） |
| `categories` | ❌ | 分类（数组格式） |
| `cover` | ❌ | 封面图路径 |
| `excerpt` | ❌ | 文章摘要 |

## 博客名称修改

编辑根目录的 `_config.yml`：

```yaml
title: 博客标题              # 浏览器标签显示
subtitle: '副标题'           # 首页副标题
author: 作者名
```

导航栏标题编辑 `_config.fluid.yml`：

```yaml
navbar:
  blog_title: "导航栏显示的文字"
```

## 配置文件位置

| 文件 | 用途 |
|------|------|
| `_config.yml` | 站点主配置（标题、URL、部署等） |
| `_config.fluid.yml` | 主题配置（导航栏、背景、评论等） |
| `scaffolds/post.md` | 新文章模板 |

## 常用命令

```bash
npx hexo new post "标题"     # 新建文章
npx hexo new draft "标题"    # 新建草稿
npx hexo server              # 本地预览
npx hexo generate            # 构建
npx hexo deploy              # 部署到 GitHub Pages
npx hexo clean               # 清理缓存
```

## 部署流程

```
写文章 → hexo generate 构建 → hexo deploy 部署 → GitHub Pages 自动更新
```

- 源码分支：`main`
- 部署分支：`gh-pages`

修改后执行：

```bash
git add -A && git commit -m "更新" && git push origin main
npx hexo clean && npx hexo generate && npx hexo deploy
```
